Section 1: Naming

  • (Section 1.1) Clear Names: Use names of Variables/Classes/Functions that explain what things do, like calling a method "CalculateTotal" instead of "Calc."
  • (Section 1.2) Naming Rules: Stick to naming rules like starting class names with big letters (like "PlayerController") and using small letters to start variables (like "playerHealth").
  • (Section 1.3) No Magic Numbers or Words: Instead of using random numbers or words in your code, create variables to hold them. For example, instead of writing "enemySpeed = 5," use "const float EnemySpeed = 5;" then use "EnemySpeed" throughout your code.

    • Section 2: Functional Code Design

  • (Section 2.1) Divide Your Code: Break your code into smaller parts that do one thing well. This makes it easier to understand and reuse. (Break your code into Functions as many as possible)
  • (Section 2.2) Unity C# Update Method: Always remember that the Update method affects FPS deeply. When editing Update, always add a time-pause.
  • (Section 2.3) Mind FPS Impact: Always remember that the Update method directly affects the Frames Per Second (FPS) of your game. When making changes to the Update method, consider adding a time-pause check to avoid unnecessarily high frame rates, especially when the game is paused. This helps in optimizing performance and preventing excessive resource consumption.
  • (Section 2.2) Use Comments Wisely: Write simple comments only when you need to explain something tricky. But try to make your code so clear that it doesn't need many comments.
  • Sensitive-Data-Storage: Don't directly save sensitive data like passwords etc., before saving sensitive data you should contact Muhammad Bin Khald .
  • Review: Always Review your code.
